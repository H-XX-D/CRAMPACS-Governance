# Audit Procedure

**Document ID:** CRAMPS-AUD-PROC-001  
**Version:** 0.1  
**Status:** Draft procedure  

## 1. Purpose

This procedure defines how CRAMPS audits are planned, executed, reported, and closed.

## 2. Auditor Independence

The lead auditor must not be the owner of the package being audited.

Auditors must disclose:

- role in study
- sponsor relationship
- financial or organizational conflict
- prior authorship of package evidence

## 3. Audit Plan

Every audit plan must define:

- audit objective
- scope
- criteria
- sample size
- evidence request list
- interview list
- schedule
- auditor names
- conflict disclosures

## 4. Evidence Request List

Minimum:

- charter
- protocol lock
- candidate registry
- source catalog
- exclusion log
- raw rows
- normalized rows
- independence groups
- bias assessment
- null model runs
- sensitivity results
- checksum manifest
- assurance case register
- CAPA log
- decision memo

## 5. Chain of Custody

Auditors must record:

- file path or system source
- timestamp accessed
- hash when available
- provider
- whether evidence is original, copy, or screenshot

## 6. Finding Lifecycle

1. Draft finding.
2. Classify severity.
3. Cite objective evidence.
4. Request management response.
5. Link CAPA if needed.
6. Verify closure.
7. Close finding.

Use `AUDIT_REPORT_TEMPLATE.md`.
