# 90-Day CRAMPACS Implementation Roadmap

## Days 0-15: Program Setup

- Appoint program owner and method owner.
- Select first domain.
- Train core roles.
- Adopt control catalog.
- Set repository and records location.
- Define data classification and retention.
- Run one tabletop preflight.

Exit criteria:

- roles assigned
- training complete
- first preflight package produced

## Days 16-45: Pilot

- Select a narrow coordinate family.
- Run `crampacs-*` preflight.
- Convert accepted preflight artifacts into `CRAMPACS-*` pilot.
- Lock protocol.
- Extract bounded sources.
- Run independence and bias review.
- Run sidecar and checksum manifest.

Exit criteria:

- pilot reaches Gate 5 or is stopped for documented reasons

## Days 46-75: Full Package Trial

- Complete null model.
- Run sensitivity tests.
- Build assurance case.
- Run clean reproduction.
- Red-team package.
- Close CAPA findings.

Exit criteria:

- full package reaches release gate or is demoted

## Days 76-90: Institutionalization

- Review pilot failure modes.
- Update templates.
- Finalize training.
- Define standing intake process.
- Define governance board cadence.
- Decide whether to expand to another domain.

Exit criteria:

- documented management review
- approved next-domain or next-study plan
