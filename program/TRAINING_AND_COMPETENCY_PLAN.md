# Training and Competency Plan

**Document ID:** CRAMPACS-TRN-001

## 1. Purpose

A CRAMPACS program is only credible if the people applying it understand the controls.

## 2. Required Training by Role

| Role | Required training |
|---|---|
| Program owner | assurance levels, decision authority, claim boundary |
| Domain lead | domain overlay, coordinate ontology, domain bias |
| Extractor | source trace, row contracts, raw/normalized separation |
| Independence auditor | evidence-family mapping, grades A-E |
| Bias auditor | missing evidence, reporting bias, domain-specific bias |
| Statistical lead | primary statistic, null model, multiplicity, sensitivity |
| Data steward | provenance, checksums, retention, data rights |
| Reproducibility lead | environment, clean run, manifest |
| Red-team reviewer | failure modes, rebuttal testing |
| Safety supervisor | evidence tiers, stop rules, decision memo |

## 3. Competency Checks

Minimum:

- classify three example rows
- identify null evidence
- grade dependence for a sample source graph
- identify coordinate/tolerance creep
- detect one unit-conversion trap
- assign correct evidence tier
- reject unsupported claim language

## 4. Recertification

Required:

- after major SOP revision
- after Critical deviation
- every 12 months for active programs
- before serving as lead reviewer in a regulated deployment
