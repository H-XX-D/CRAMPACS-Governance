## 1. Title

Identify the study as a CRAMPS protocol and state the coordinate family and domain.

## 2. Rationale

State why this coordinate-resolved recurrence inspection is being performed. Include what is known, what is uncertain, and why isolated review of single findings is insufficient.

## 3. Objectives

Define the primary question, secondary questions, and negative controls.

## 4. Candidate Coordinates

Reference `candidate_coordinate_registry.csv`. State how candidates were selected before scoring.

## 5. Eligibility Criteria

Define included and excluded source types, row types, date ranges, languages, publication statuses, and required metadata.

## 6. Information Sources

List databases, collaboration pages, archives, datasets, reviews, conference note sources, and search dates.

## 7. Search Strategy

Provide exact search strings and source-specific procedures.

## 8. Screening Procedure

Define reviewer count, disagreement resolution, and exclusion reason taxonomy.

## 9. Extraction Procedure

Reference `anomaly_rows_raw.csv` as the stable raw-signal table name. Define extraction fields, plot digitization policy, and confidence scoring.

## 10. Coordinate Normalization

Reference `coordinate_transform_registry.csv`. Define canonical units, allowed transforms, uncertainty propagation, and forbidden transforms.

## 11. Independence Assessment

Reference `independence_groups.csv`. Define evidence families, dependence grades, and primary weights.

## 12. Bias and Missing Evidence Assessment

Reference `bias_assessment.csv`. Define publication bias, selective reporting, missing-null risk, analysis flexibility risk, coordinate-fashion risk, and extraction bias.

## 13. Statistical Analysis Plan

Define the primary statistic, secondary statistics, null models, random seeds, simulation counts, multiple-testing correction, and uncertainty reporting.

## 14. Sensitivity Tests

Define leave-source-out, leave-experiment-out, leave-modality-out, leave-era-out, extraction-confidence, bias, tolerance, and null-model sensitivity analyses.

## 15. Negative Controls

Reference negative control coordinates and expected behavior under the null.

## 16. Reporting Plan

Define report sections, figures, tables, evidence tiers, and claim language restrictions.

## 17. Reproducibility Plan

Define code package, dependency lock, checksums, clean-run procedure, and data availability.

## 18. Amendments

Define what counts as administrative, minor, or major amendment. State how result-blind status will be recorded.

## 19. Conflicts, Funding, and Contributions

Record conflicts of interest, funding source, sponsor role, and CRediT-style contributions.
