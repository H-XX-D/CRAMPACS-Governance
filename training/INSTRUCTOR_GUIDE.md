# Instructor Guide

**Document ID:** CRAMPS-TRN-INST-001  
**Version:** 0.1  
**Status:** Draft instructor guide

## 1. Instructor Goal

Teach CRAMPS as a controlled decision system, not as an exciting anomaly story.

The instructor should repeatedly anchor learners to three questions:

1. What coordinate was locked?
2. What null or non-event evidence was included?
3. What decision is justified by this evidence tier?

## 2. Teaching Principles

| Principle | Instructor behavior |
|---|---|
| Make boundaries visible | Correct overclaiming immediately |
| Teach by artifacts | Use templates and registers, not slides alone |
| Use stop signs | Reward learners for holding or rejecting weak packages |
| Separate evidence from action | Make learners state both tier and decision |
| Make uncertainty concrete | Require missing-evidence and dependence notes |

## 3. Module Facilitation Notes

### Module 1: Method and boundary

Emphasize:

- CRAMPS is not a discovery shortcut.
- Lowercase preflight is not full assurance.
- A result can be interesting and still not releasable.

Ask:

- What would be an overclaim in your domain?
- Who could be harmed by overclaiming?

### Module 2: Coordinate lock

Demonstrate:

- vague coordinate
- locked coordinate
- tolerance creep
- forbidden post-hoc change

Ask learners to rewrite a vague target until it can be audited.

### Module 3: Source universe

Demonstrate:

- positive-only package
- null-inclusive package
- source flow with exclusions

Ask:

- What source would you least want to find?
- What source would most challenge the pattern?

### Module 4: Dependence and bias

Demonstrate:

- five rows from one dataset
- five rows from five independent source families
- hidden dependence through calibration, vendor, model, or source system

Ask learners to draw a source-family map before scoring.

### Module 5: Statistical method

Do not over-teach math. Teach method control:

- primary statistic locked before scoring
- null model represents the source process
- local results need global correction
- negative controls test the pipeline

### Module 6: Sidecar and release

Show that sidecar blockers are not failures. They are control signals.

Make learners defend one of:

- continue
- hold
- stop
- demote
- escalate

## 4. Common Learner Mistakes

| Mistake | Correction |
|---|---|
| Treating preflight as proof | Restate lowercase/uppercase boundary |
| Searching only for interesting anomalies | Require null and non-event evidence |
| Counting duplicate rows independently | Require evidence-family map |
| Moving coordinate after seeing data | Trigger coordinate-lock failure |
| Reporting local recurrence as final | Require null model and global correction |
| Ignoring missing evidence | Open bias issue or hold release |
| Treating attendance as competence | Require scored exercise |

## 5. Instructor Debrief Script

Use this after exercises:

1. What decision did your team make?
2. What is the evidence tier?
3. Which evidence most supports the decision?
4. Which evidence most weakens the decision?
5. What would force a hold?
6. What claim is prohibited?
7. What is the next controlled action?

## 6. Minimum Teaching Kit

Before class, prepare:

- one mock domain scenario
- one positive-only evidence set
- one null-inclusive evidence set
- one duplicate-evidence trap
- one sidecar output with blockers
- one decision memo example
- one CAPA example
